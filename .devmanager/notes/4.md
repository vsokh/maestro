## Task description
The orchestrator reads specs from .devmanager/specs/, but the manager has no way to write them from the UI. Add a markdown editor in the detail panel so the manager can write detailed specs for complex tasks.

## Exploration findings
- **TaskDetail.jsx** — Detail panel with inline name editing, notes textarea, screenshots, dependencies, action buttons. All inline styles using CSS vars.
- **fs.js** — File System Access API functions: readState/writeState, saveAttachment/deleteAttachment/readAttachmentUrl. Good pattern to follow for specs.
- **App.jsx** — Central state management. `handleUpdateNotes` saves to `taskNotes` in state.json. TaskDetail receives `notes` prop + `onUpdateNotes` callback + `dirHandle`.
- **No existing spec functions** — nothing in fs.js for reading/writing specs yet.
- **.devmanager/specs/` directory referenced in CLAUDE.md and orchestrator.js but never created by the UI.

## Proposed plan

### 1. Add fs.js functions (~20 LOC)
- `writeSpec(projectHandle, taskId, content)` — writes to `.devmanager/specs/{taskId}.md`
- `readSpec(projectHandle, taskId)` — reads `.devmanager/specs/{taskId}.md` as text
- `deleteSpec(projectHandle, taskId)` — removes the spec file

Use `{taskId}.md` as filename for simplicity. The orchestrator can find specs by task ID.

### 2. Add spec handling to App.jsx (~25 LOC)
- Track `specContent` state per selected task (load on task select)
- `handleSaveSpec(taskId, content)` — calls writeSpec, updates local state
- `handleDeleteSpec(taskId)` — calls deleteSpec, clears local state
- Pass `spec`, `onSaveSpec`, `onDeleteSpec` as props to TaskDetail

### 3. Add spec editor section to TaskDetail.jsx (~60 LOC)
- New section between "Notes for Claude" and "Screenshots", labeled **"Spec"**
- **Empty state**: subtle dashed-border prompt "Add a detailed spec..."
- **Editing**: Textarea (taller than notes, ~8 rows) with Save/Discard buttons
- **View mode**: Rendered as preformatted text (no markdown rendering library — keep deps minimal)
- Toggle between view/edit on click, similar to inline name editing pattern

### Files to modify
- `src/fs.js` — add writeSpec, readSpec, deleteSpec
- `src/App.jsx` — add spec state, handlers, pass props to TaskDetail
- `src/components/TaskDetail.jsx` — add spec editor section
