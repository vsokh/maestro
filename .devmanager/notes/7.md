# Task 7: Feature-task linking

## Manager notes
(none provided)

## Task description
Features exist but aren't connected to tasks. Let the manager group tasks under a feature (epic) to get a higher-level view of progress per feature.

## Exploration findings
- Tasks already have a `group` field linking to epics (string match on epic name)
- Epics registry exists in state.json with `name` + `color` fields
- `features` array in state.json is always empty — never populated or used
- TaskBoard groups tasks by epic with colored headers (EPIC_PALETTE, 8 colors)
- TaskDetail has an epic assignment input with datalist autocomplete
- CardForm allows setting epic at task creation time
- CommandQueue groups queue items by epic

### What's missing (the actual task)
- **No progress tracking per epic** — no "X of Y done" anywhere
- **No epic overview/summary** — no way to see all epics at a glance with progress
- **No epic color indicator** in TaskDetail or CardForm (just plain text input)
- The `features` array is dead code / vestigial

## Proposed plan

### 1. Add epic progress counters to group headers in TaskBoard
- On each epic group header in the "Up next" section, show progress like "Core 3/5"
- Count includes ALL tasks in that epic (pending + done + in-progress + paused + blocked)
- This gives the "higher-level view of progress per feature"

### 2. Add Epic Overview section at top of TaskBoard
- Compact row of colored epic pills, each showing: name + "X/Y done" + mini progress bar
- Clicking an epic pill could scroll to or highlight that group
- Gives a dashboard-level summary of feature progress

### 3. Show epic color indicator in TaskDetail
- Add a small colored dot/badge next to the epic input field matching the epic's palette color
- Visual feedback connecting the text input to the colored groups on the board

### 4. Clean up unused `features` array
- Remove `features` from state.json references since epics serve the same purpose
- Already empty and unused
