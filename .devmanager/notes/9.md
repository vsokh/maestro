# Task 9: State backup and undo

## Manager notes
No way to recover from accidental task deletes or bad edits. Auto-save a snapshot of state.json before destructive operations (delete, bulk changes) and provide a simple undo/restore mechanism.

## Exploration findings
- State written via `writeState()` in `src/fs.js` (lines 70-79), uses File System Access API
- All state mutations flow through `updateData()` in `src/App.jsx` (lines 95-98)
- 500ms debounced save in `useProject.js` (lines 127-139)
- 3s polling for external changes in `useProject.js` (lines 142-244)
- No backup/undo mechanism exists anywhere
- Destructive ops: delete task (App.jsx:139), clear queue (App.jsx:209), cancel task (useProject.js:278), delete attachment (App.jsx:166), remove activity (App.jsx:277), remove from queue (App.jsx:204)
- Only delete task has a confirmation dialog
- `.devmanager/backups/` directory does not exist yet

## Proposed plan
1. Add snapshot/restore/prune/list functions to `src/fs.js`
2. Add in-memory undo stack + snapshot calls before destructive ops in `src/App.jsx`
3. Create `src/components/UndoToast.jsx` — floating undo notification
